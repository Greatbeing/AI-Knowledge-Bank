# AI Knowledge Bank 贡献指南

欢迎参与 AI Knowledge Bank 项目！我们致力于构建 AI 时代的人类技能进化网络。

## 🌟 如何贡献

### 1. 报告问题 (Bug Reports)
发现 Bug 或有改进建议？请创建 [Issue](https://github.com/yourusername/ai-knowledge-bank/issues)。

**提交 Bug 时请包含：**
- 清晰的标题和描述
- 复现步骤
- 预期行为 vs 实际行为
- 环境信息（浏览器版本、操作系统等）
- 截图或录屏（如适用）

### 2. 功能建议 (Feature Requests)
有新想法？我们很乐意听取！

**提交建议时请说明：**
- 功能描述和使用场景
- 为什么这个功能重要
- 可能的实现思路（可选）

### 3. 代码贡献 (Code Contributions)

#### 开发环境设置
```bash
# Fork 并克隆仓库
git clone https://github.com/YOUR_USERNAME/ai-knowledge-bank.git
cd ai-knowledge-bank

# 创建特性分支
git checkout -b feature/your-feature-name
```

#### 代码规范
- 使用语义化的变量和函数命名
- 添加必要的注释说明复杂逻辑
- 保持代码格式整洁（推荐使用 Prettier）
- 确保新功能有适当的测试（如适用）

#### 提交规范
遵循 [Conventional Commits](https://www.conventionalcommits.org/) 规范：

```bash
# 示例
git commit -m "feat: 添加用户声誉系统"
git commit -m "fix: 修复演化树动画闪烁问题"
git commit -m "docs: 更新 README 安装说明"
git commit -m "refactor: 重构节点渲染逻辑"
```

**类型说明：**
- `feat`: 新功能
- `fix`: Bug 修复
- `docs`: 文档更新
- `style`: 代码格式调整
- `refactor`: 代码重构
- `test`: 测试相关
- `chore`: 构建/工具链相关

#### 提交 PR
1. 推送分支到远程仓库
   ```bash
   git push origin feature/your-feature-name
   ```
2. 在 GitHub 上创建 Pull Request
3. 填写详细的 PR 描述，说明变更内容和原因
4. 等待 Code Review 和合并

### 4. 内容贡献 (Content Contributions)

作为知识银行，我们特别欢迎内容贡献：

- **分享实战 SOP**: 提交你的 AI 工作流和 Prompt 模板
- **编写教程**: 针对特定场景的使用指南
- **翻译文档**: 帮助项目国际化
- **案例研究**: 真实业务场景的应用案例

内容可以提交到 `content/` 目录或通过 Issue 提供。

## 📜 行为准则

我们致力于提供一个开放、友好、包容的社区环境。所有参与者应遵守：

- 尊重不同观点和经验
- 优雅地接受建设性批评
- 关注对社区最有利的事情
- 对其他社区成员表示同理心

任何形式的骚扰、歧视或不当行为都是不可接受的。
